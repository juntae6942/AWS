# AWS
### 클라우드 컴퓨팅이란?  
인터넷을 통해 IT 리소스와 애플리케이션을 온디맨드로 제공하는 서비스
즉, 사용한만큼 비용을 지불하는 서비스
- 온디맨드 서비스 액세스
- 대규모 사전 투자 방지 
- 필요에 따라 컴퓨팅 리소스 프로비저닝 
- 사용한 만큼만 비용을 지불

### 클라우드  
모든 부분을 클라우드에서 실행  
### 온프레미스
가상화 및 리소스 관리 도구를 사용하여 리소스 배포  
애플리케이션 관리 및 가상화 기술을 사용  
### 하이브리드  
클라우드 + 온프레미스
### 클라우드 컴퓨팅의 이점
- 초기 비용
  - 기술 리소스를 사용하기 전에 먼저 투자
- 가변 비용
  - 사용한만큼만 비용을 지불
- 비용 최적화
  - 구축 시간 증가
    - 애플리케이션과 고객에 집중
  - 비용 관리에 대한 시간 절약
    - 데이터 센터 운영
- 용량
  - 필요한 용량만 액세스
  - 확장성
- 규모의 경제
  - 집계된 고객 사용량으로 인한 이점
- 속도 및 민첩성
  - 클라우드 컴퓨팅
    - 빠른 시간내에 필요한 리소스 얻을 수 있음
- 글로벌 인프라

### EC2
크기 조정이 가능한 컴퓨팅 용량을 제공하는 인스턴스
인스턴스 유형
- 범용
  - 다양한 워크로드에 적합
  - 컴퓨팅, 메모리, 네트워킹 리소스를 균형 있게 제공
- 컴퓨팅 최적화
  - 고성능 프로세서 제공
  - `컴퓨팅 집약적 애플리케이션 및 배치 처리 워크로드`에 적합
- 메모리 최적화
  - 메모리 집약적 워크로드를 위한 빠른 성능 제공
  - `고성능 데이터베이스`에 적합
- 가속 컴퓨팅
  - 하드웨어 액셀레이터를 사용하여 **`데이터 처리 가속화`**
  - 애플리케이션 스트리밍 및 그래픽 워크로드에 적합
- 스토리지 최적화
  - **`낮은 지연 시간 및 높은 초당 입출력`** 작업 수(IOPS)제공
  - **`분산 파일 시스템`** 및 **`데이터 웨어하우징`** 애플리케이션과 같은 워크로드에 적합
#### 요금 옵션
- 온디맨드
  - 초기 선결제 비용이나 최소 약정 없음
  - 불규칙한 단기 워크로드에 적합
- 스팟
  - 시작 및 종료 시간이 자유로운 워크로드에 적합
  - 온디맨드 대비 비용 절감
  - 평균 5%의 확률로 사라질 수 있다.`(3개월에 한번정도)`
- 예약
  - 온디맨드 요금에 비해 `결제 할인 제공`
  - 1년 또는 3년 약정 필요
- Compute Savings Plans
  - 컴퓨팅 사용량이 일정할 경우 온디맨드에 비해 최대 66%비용 절감
  - 1년 또는 3년 약정 필요
- 전용 인스턴스
  - 단일 고객용 하드웨어의 VPC에서 실행되는 EC2 인스턴스
  - 표준 Amazon EC2 인스턴스에 비해 더 높은 비용
- 전용 호스트
  - 단일 고객용 EC2 인스턴스 용량을 갖춘 `물리적 서버`
  - 가장 비용이 많이 드는 Amazon EC2 옵션
  - `Tenancy` 부분에서 설정

### Auto Scaling
동적 크기 조정
- 수요변화에 대응

에측 크기 조정
- `예측된 수요`에 따라 적절한 수의 EC2 인스턴스를 자동으로 예약  

**함께 사용하면 더 빠르게 조정 가능**

### Load Balancing
- 트래픽 분산
- 웹 트래픽의 단일 접점 역할을 한다.
- Auto Scaling과 별도의 서비스지만 연동 가능
- 고성능, 고가용성을 제공

### Server Less Lambda
- 서버를 프로비저닝하거나 관리하지 않고 코드 실행
- 코드가 실행되는 컴퓨팅 시간에 대해서만 비용 지불
- 다른 AWS 서비스를 사용하여 코드를 자동으로 트리거

### `ECS(Elastic Container Service)`
- 컨테이너화된 애플리케이션 실행 및 크기 조정
- 간단한 API 호출을 사용하여 Docker 지원 애플리케이션 제어
- 컨테이너 오케스트레이션(관리 및 배포) 서비스
### **`EKS(Elastic Kubernetes Service)`**  
- Kubernetes 애플리케이션 실행 및 크기 조정
- 새로운 기능으로 애플리케이션을 손쉽게 업데이트
- 컨테이너 오케스트레이션(관리 및 배포) 서비스
### Fargate
- 컨테이너용 서버리스 컴퓨팅 엔진(ECS, EKS에서 작동)
- ECS 또는 EKS를 사용하여 서버리스 컨테이너 실행
- 사용한 리소스에 대해서만 비용 지불
- 컨테이너 호스팅 플랫폼
- 자동으로 서버 인프라 관리

### Outposts 
온프레미스 위치 내에서 하이브리드 방식으로 AWS 인프라, 서비스, 도구를 실행하는데 사용할 수 있는 서비스

### VPC(Virtual Private Cloud)
가상 네트워크  
#### 서브넷 
- 격리된 리소스 그룹을 배치할 수 있는 VPC의 섹션
- 퍼블릭 서브넷
- 프라이빗 서브넷  

클라이언트가 인터넷을 통해 VPC로 요청 전송 -> 인터넷 `게이트웨이`를 통해 접속  
`가상 프라이빗 게이트웨이` - VPC 내의 프라이빗 리소스에 액세스  
`Direct Connect` 데이터 센터와 VPC 간에 전용 프라이빗 연결을 설정하는 서비스  

### NACL(Network Access Control List)  
- 서브넷 수준에서 인바운드 및 아웃바운드 트래픽을 제어하는 `가상 방화벽`  
- 계정의 기본 NACL은 기본적으로 모든 인바운드 및 아웃바운드 트래픽을 허용  
- 사용자 지정 NACL은 모든 인바운드 및 아웃바운드 트래픽을 거부  
- `보안 그룹은 상태 저장(Stateful)` 패킷 필터링을 수행  
- 보안 그룹은 들어오는 패킷에 대해 내린 이전 결정을 기억  

### 보안그룹
- **EC2 인스턴스의 가상 방화벽**
- 모든 인바운드 트래픽을 거부
- 모든 아웃바운드 트래픽을 허용  

**보안그룹은 반드시 지정, NACL은 선택적**  
- NACL은 stateless 패킷 필터링
- 보안그룹은 stateful 패킷 필터링

### Route53
DNS 웹 서비스  
사용자의 요청을 AWS에서 실행되는 인프라 인스턴스 및 로드 밸런서에 연결  
사용자를 AWS 외부의 인프라로 라우팅 할 수도 있음

### 블록 스토리지
- 인스턴스 스토어
  - 임시 블록 수준 스토리지
  - EC2 종료하면 모든 데이터 삭제

- EBS(Elastic Block Store)
  - EC2 인스턴스에서 사용할 수 있는 블록 수준 스토리지
  - EC2 종료해도 모든 데이터 유지
  - EBS 스냅샷을 생성하여 EBS 볼륨을 증분 백업할 수 있다.  

### 객체 스토리지(S3, Simple Storage Service)
- 데이터
- 메타 데이터
- 키
- 전송요금이 있음

### RDS(Relational Database Service)
- 저장시 암호화 및 전송 중 암호화를 제공
### DynamoDB
- 비관계형 데이터베이스  
- 일관된 성능 유지
- NoSQL(Not only SQL)
### Aurora
- AWS에서 제공하는 오픈소스 DB, MySQL, PostgreSQL 호환
### DMS(Database Migration Service)
- RDB, NoRDB 및 기타 유형의 데이터 스토어를 마이그레이션
- 원본 DB와 대상 DB 사이에 데이터 이동 시 사용
- 데이터베이스 통합
- 지속적 복제
### 추가 데이터베이스 서비스
Redshift  
- 데이터 웨어하우스에서 데이터를 쿼리 및 분석

DocumentDB  
- 문서 데이터베이스 서비스에서 MongoDB 워크로드를 실행

Neptune  
- 고도로 연결된 데이터세트를 사용하는 애플리케이션을 실행  

QLDB  
- 애플리케이션 데이터의 전체 변경 내역을 검토  

### IAM 정책
AWS 서비스 및 리소스에 대한 권한을 허용하거나 거부하는 문서 `최소 권한 원칙을 준수`

### MFA(다중인증)
### Artifact
필요에 따라 보안 및 규정 준수 보고서와 일부 온라인 계약에 액세스

### WAF
악성 트래픽 차단  
웹 애플리케이션 방화벽  
CloudFront 통합 - CloudFront에서 WAF를 사용하면 전 세계에서 최종 사용자와 가까운 AWS 엣지 로케이션에서 규칙이 실행  
보안을 위해 성능을 희생할 필요가 없음  
차단된 요청은 고객 웹 서버에 도달하기 전에 중지됨  
LoadBalancer에서 WAF를 사용하면 규칙이 리전에서 실행되며, 규칙을 사용해 인터넷과 연결된 로드 밸러서와 내부 로드 밸런서를 보호할 수 있다.  
- 허용 목록
- 거부 목록
### Shield
DDos 공격으로부터 애플리케이션을 보호하는 서비스
- Standard 및 Advanced
### Inspector
자동화된 보안 평가  
보안 및 규정 준수를 개선할 수 있다.
EC2 인스턴스에 대한 오픈 액세스, 취약한 소프트웨어 버전 설치와 같은 보안 모범 사례 위반 및 보안 취약성을 검사
